# Acquisition seq8 logging test

Firmware: `dev-2026.06.13.18`

## Goal

Add one 8-bit acquisition counter on Core 0 so we can check if any sensor data capture is missing. The counter is logged with the sensor timestamp for mic, accelerometer and ECG.

## Change

The USB and WiFi CSV log now includes these fields after the row timestamp:

```text
mic_ms,mic_seq8,acc_ms,acc_seq8,ecg_ms,ecg_seq8
```

The `*_ms` fields are the timestamp from the sensor acquisition frame. The `*_seq8` fields come from one shared Core 0 8-bit counter. It increments when Core 0 queues each acquired sensor frame and wraps from `255` back to `0`.

This means the log can show delayed or missing frames. If a sensor sequence jumps more than expected, we know data capture or queue handling needs checking.

## Verification

- Built firmware with PlatformIO successfully.
- Flashed `dev-2026.06.13.18` to the device on `COM14`.
- Serial boot log confirmed `device version: dev-2026.06.13.18`.
- Took a 5 second USB live log from `COM14`.
- Parsed 243 valid CSV rows.
- Confirmed all new fields are present: `mic_ms`, `mic_seq8`, `acc_ms`, `acc_seq8`, `ecg_ms`, `ecg_seq8`.
