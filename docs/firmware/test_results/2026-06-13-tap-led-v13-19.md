# Accelerometer tap LED test

Firmware: `dev-2026.06.13.19`

## Goal

Use the accelerometer as a simple tap input while the device is not logging.

## Behaviour

- Single tap toggles the blue LED on GPIO15.
- Next single tap toggles it off.
- Tap detection is ignored while USB or WiFi logging is active.
- During logging, the blue LED remains the logging-active indicator.
- Tap detection runs after accelerometer frames are drained on Core 1, so Core 0 stays acquisition-only.

## Implementation

Core 1 calculates a jerk-style delta from the latest valid accelerometer axes. If the delta is above the tap threshold and outside the refractory window, the firmware toggles `blueTapLedOn` and prints:

```text
TAP,ms=<time>,delta=<g>,blue=<0/1>
```

## Verification

- Built firmware with PlatformIO successfully.
- Flashed `dev-2026.06.13.19` to the device on `COM14`.
- Serial boot log confirmed `device version: dev-2026.06.13.19`.
- QMI8658 accelerometer initialized successfully at `0x6B`.
- Watched serial for 25 seconds. No physical tap was captured in that window, so no `TAP` line was observed from the remote test.
